# CSV 目录结构

## 目录规范

CSV 测试案例文件必须放在特定的目录结构下才能被插件识别。

### 标准目录结构

```
工作区根目录/
└── 测试任务/                    # 或 testtask (英文)
    └── 测试任务名称_子任务名称/   # 例如: 任务1_登录模块
        └── 测试案例/            # 或 testcase (英文)
            ├── testcases.csv    # 测试案例文件
            ├── testcases1.csv
            └── ...
```

### 支持的目录名称

| 中文目录名 | 英文目录名 | 说明 |
|-----------|-----------|------|
| 测试任务 | testtask | 顶层任务目录 |
| 测试案例 | testcase | 包含案例文件的目录 |

### 支持的文件类型

| 文件类型 | 说明 |
|---------|------|
| `*.csv` | CSV 格式测试案例 |
| `*.yaml` / `*.yml` | YAML 格式测试案例 |
| `*.json` | JSON 格式测试案例 |

### 完整示例

```
项目根目录/
├── 业务代码/
│   └── ...
└── 测试任务/
    ├── 登录模块测试_正常场景/
    │   └── 测试案例/
    │       └── testcases.csv
    ├── 登录模块测试_异常场景/
    │   └── 测试案例/
    │       ├── testcases.csv
    │       └── testcases_backup.csv
    └── testtask/
        └── task_regression/
            └── testcase/
                └── regression.csv
```

### 路径验证规则

插件会检查文件路径是否满足以下条件：
1. 文件协议为 `file://`
2. 目录深度至少为 4 层
3. 倒数第 4 层目录名为 `测试任务` 或 `testtask`
4. 倒数第 2 层目录名为 `测试案例` 或 `testcase`
5. 文件扩展名匹配对应的编辑器类型
