# 测试案例目录结构

## 目录规范

CSV / YAML / JSON 测试案例文件必须放在特定的目录结构下才能被插件识别。文件可直接放在 `测试案例/` 目录下，也可放在其子目录中。

### 标准目录结构

```
工作区根目录/
└── 测试任务/                              
    └── <testTaskNo>_<subTestTaskName>/    # 例如: TT001_登录模块
        └── 测试案例/                     
            ├── testcases.csv              # 测试案例文件（可直接放此目录）
            ├── module_a/                  # 也可按模块分子目录
            │   ├── testcases.csv
            │   └── testcases.yaml
            ├── module_b/
            │   └── testcases.json
            └── ...
```

### 第三层目录命名约定

第三层目录名由 **测试任务编号** 与 **测试子任务名称** 通过下划线 `_` 拼接：

```
<testTaskNo>_<subTestTaskName>
```

| 示例目录 | testTaskNo | subTestTaskName |
|---------|-----------|-----------------|
| `TT001_登录模块` | `TT001` | `登录模块` |
| `TT2026010101_风控_主流程` | `TT2026010101` | `风控_主流程` |

> 切分规则：按 **第一个** `_` 切分。前段为 `testTaskNo`，剩余部分（保留可能含有的下划线）整体作为 `subTestTaskName`。

### 推送时的字段解析

推送测试案例（无论是右键文件菜单还是编辑器内"推送"按钮）时，插件会从当前文件路径中按照上述约定解析出 `testTaskNo` 与 `subTestTaskName`，与所选数据一同提交给后端：

请求路径：`POST /test-task/push-testcase`

请求 body 结构：

```json
{
    "testTaskNo": "TT001",
    "subTestTaskName": "登录模块",
    "data": []
}
```

> `data` 字段为选中行组成的数组（具体结构与文件类型一致）。

若目录不符合上述命名约定，推送会被拒绝并提示：
> 无法从路径解析测试任务信息，请确认目录形如：测试任务/<编号>_<子任务名>/测试案例/<文件>

### 支持的目录名称

| 中文目录名 | 英文目录名 | 说明 |
|-----------|-----------|------|
| 测试任务 | testtask | 顶层任务目录 |
| 测试案例 | testcase | 包含案例文件的目录 |

### 支持的文件类型

| 文件类型 | 说明 |
|---------|------|
| `*.csv` | CSV 格式测试案例 |
| `*.yaml` / `*.yml` | YAML 格式测试案例 |
| `*.json` | JSON 格式测试案例 |

### 完整示例

```
项目根目录/
├── 业务代码/
│   └── ...
└── 测试任务/
    ├── TT001_登录正常场景/
    │   └── 测试案例/
    │       ├── testcases.csv               # 直接放在测试案例目录下
    │       └── 登录/                       # 分子目录管理
    │           └── login_cases.yaml
    ├── TT002_登录异常场景/
    │   └── 测试案例/
    │       ├── 输入校验/
    │       │   ├── validation.csv
    │       │   └── edge_cases.json
    │       └── 错误处理/
    │           └── error_cases.yaml
    └── T00_任务1/
        └── 测试案例/
            └── TT_REG_regression/
                ├── regression.csv
                └── 回归子模块/
                    └── sub_regression.yaml
```

### 路径验证规则

插件会检查文件路径是否满足以下条件：
1. 文件协议为 `file://`
2. 目录深度至少为 4 层（文件在 `测试案例/` 子目录下时为 5 层或更多）
3. 路径中包含 `测试任务` 或 `testtask` 目录
4. 紧随其后的子目录名形如 `<testTaskNo>_<subTestTaskName>`（含至少一个 `_`，且不以 `_` 开头或结尾）
5. 路径中包含 `测试案例` 或 `testcase` 目录（位于 `<testTaskNo>_<subTestTaskName>` 之下）
6. 文件扩展名匹配对应的编辑器类型（`.csv` / `.yaml` / `.yml` / `.json`）
