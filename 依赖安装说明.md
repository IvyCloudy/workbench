# 依赖安装说明

## 📋 概述

重构完成后，需要安装项目依赖以解决TypeScript编译错误。当前项目缺少`node_modules`目录，需要安装所有依赖包。

## 🔧 安装步骤

### 1. 安装依赖

在项目根目录下执行以下命令：

```bash
npm install
```

这将安装`package.json`中定义的所有依赖项，包括：

**生产依赖：**
- `iconv-lite` - 字符编码转换
- `sm-crypto` - 国密算法加密
- `udsv` - CSV解析器
- `xlsx` - Excel文件处理
- `yaml` - YAML解析器

**开发依赖：**
- `@types/node` - Node.js类型定义
- `@types/vscode` - VS Code API类型定义（关键）
- `typescript` - TypeScript编译器
- `vitest` - 测试框架

### 2. 验证安装

安装完成后，检查以下内容：

```bash
# 检查node_modules是否存在
ls node_modules/

# 检查vscode类型定义
ls node_modules/@types/vscode/

# 编译项目
npm run compile
```

### 3. 解决lint错误

安装依赖后，以下lint错误应该会自动解决：

- ✅ `TS2307: Cannot find module 'vscode'`
- ✅ `TS2307: Cannot find module 'path'`
- ✅ `TS2584: Cannot find name 'console'`

## 🚨 常见问题

### 问题1：网络连接问题

如果安装过程中遇到网络问题，可以尝试：

```bash
# 使用淘宝镜像
npm install --registry=https://registry.npmmirror.com

# 或使用yarn
npm install -g yarn
yarn install
```

### 问题2：权限问题

如果遇到权限错误：

```bash
# 清除npm缓存
npm cache clean --force

# 删除node_modules重新安装
rm -rf node_modules
npm install
```

### 问题3：版本冲突

如果出现版本冲突：

```bash
# 删除package-lock.json重新安装
rm package-lock.json
npm install
```
