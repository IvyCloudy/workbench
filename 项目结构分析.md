## 2. 架构概览

### 2.1 核心模块

| 模块 | 职责 |
|------|------|
| **extension.ts** | 插件入口，注册命令和自定义编辑器 |
| **providers/** | 各种编辑器和视图的提供者 |
| **services/** | 公共服务层（HTTP、存储、解析等） |
| **types/** | TypeScript 类型定义 |
| **media/** | Webview 静态资源 |

### 2.2 编辑器类型

| 编辑器 | ViewType | 文件类型 |
|--------|----------|---------|
| CSV 编辑器 | `csvEditor.testCase` | `.csv` |
| YAML 编辑器 | `yamlEditor.testCase` | `.yaml` / `.yml` |
| JSON 编辑器 | `jsonEditor.testCase` | `.json` |

### 2.3 Webview 视图

| 视图 | 提供者 | 功能 |
|------|--------|------|
| 工作台 | `WorkbenchProvider` | 主界面入口 |
| CSV 浏览器 | `CsvBrowserProvider` | CSV 文件浏览 |
| 在线案例 | `TestCaseProvider` | 查看线上测试案例 |

## 3. 服务层详解

### 3.1 `services/csv-parser.ts`

唯一的 CSV 解析来源，使用 `xlsx` 库解析。

**核心函数：**
- `loadCsvFromFile(filePath)` - 从文件加载 CSV
- `loadCsvFromContent(content)` - 从字符串加载 CSV
- `sheetToCsv(sheets)` - 导出为 CSV 格式
- `exportToCsv(sheets)` - 导出为 Excel 格式

### 3.2 `services/http-client.ts`

TypeScript 版本的 HTTP 客户端，使用原生 Node.js `http`/`https`。

**核心函数：**
- `queryApi(opts, context)` - 查询测试案例
- `fetchTaskTree(context)` - 获取任务树
- `sendSelectedData(opts, context)` - 批量导入选中数据

### 3.3 `services/storage.ts`

统一的存储服务，合并了原来的 `config.js` 和 `store.ts`。

**核心函数：**
- `readConfig(context)` - 读取应用配置
- `writeConfig(context, partial)` - 更新配置
- `readParams(context)` - 读取查询参数
- `writeParams(context, params)` - 保存查询参数

### 3.4 `services/utils.ts`

通用工具函数。

**核心函数：**
- `getNonce()` - 生成 CSP Nonce
- `escapeHtml(str)` - HTML 转义
- `isInQualifiedDir(uri, pattern)` - 检查文件路径
- `debounce(fn, delay)` - 防抖函数
- `deepClone(obj)` - 深拷贝

### 3.5 `services/table-editor-template.ts`

表格编辑器的 HTML 模板。

## 6. 扩展入口：`src/extension.ts`

```typescript
export function activate(context: vscode.ExtensionContext) {
    // 创建 Provider 实例
    const workbenchProvider = new WorkbenchProvider(context.extensionUri, context);
    const csvBrowserProvider = new CsvBrowserProvider(context.extensionUri, context);
    const csvEditorProvider = new CsvEditorProvider(context.extensionUri, context);
    const yamlEditorProvider = new YamlEditorProvider(context.extensionUri, context);
    const jsonEditorProvider = new JsonEditorProvider(context.extensionUri, context);

    // 注册自定义编辑器
    context.subscriptions.push(
        vscode.window.registerCustomEditorProvider('csvEditor.testCase', csvEditorProvider),
        vscode.window.registerCustomEditorProvider('yamlEditor.testCase', yamlEditorProvider),
        vscode.window.registerCustomEditorProvider('jsonEditor.testCase', jsonEditorProvider)
    );

    // 注册命令
    // ... (省略详细代码)

    // 激活时显示工作台
    workbenchProvider.show();
```

## 10. 架构优势

1. **类型安全** - 全面使用 TypeScript，类型定义集中管理
2. **代码复用** - BaseProvider 基类减少重复代码
3. **单一数据源** - CSV 解析统一使用 `csv-parser.ts`
4. **模块化** - 服务层和 Provider 层分离
5. **可测试** - 完整的单元测试覆盖
6. **清晰结构** - 每层职责明确，易于维护
